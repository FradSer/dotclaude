# Sections

This file defines all sections, their ordering, impact levels, and descriptions.
The section ID (in parentheses) is the filename prefix used to group rules.

---

## 1. Fundamental Rules (fundamental)

**Impact:** CRITICAL  
**Description:** Core principles regarding imports, exceptions, resource management, and state. Violations here can lead to serious bugs or maintainability nightmares.

## 2. Code Quality & Safety (quality)

**Impact:** HIGH  
**Description:** Practices that ensure code robustness, type safety, and maintainability (linting, types, function size).

## 3. Language Features (language)

**Impact:** MEDIUM-HIGH  
**Description:** Idiomatic usage of Python features like comprehensions, decorators, and generators. Proper use ensures "Pythonic" code.

## 4. Formatting & Style (style)

**Impact:** MEDIUM  
**Description:** Visual consistency rules (naming, spacing, strings). Important for readability but less critical for correctness.
