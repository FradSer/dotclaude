# Task 006: Create sprint contract template reference

**depends-on**: task-002

## Description

Create a Level 3 reference file providing the template and guidance for sprint contract files. Sprint contracts are produced per-batch by the evaluator before execution begins, replacing implicit done criteria with concrete, testable checklists. The template must handle Red-Green pair distinction and ambiguity flagging.

## Execution Context

**Task Number**: 006 of 10
**Phase**: Core Features (REQ-002)
**Prerequisites**: Task 002 (file formats define the canonical sprint contract structure)

## BDD Scenario

```gherkin
Scenario: Sprint contract produced per-batch before execution
  Given a batch with tasks containing BDD scenarios
  And the evaluator agent is enabled
  When the batch is about to begin execution (Phase 3, step 0)
  Then a sprint-contract-batch-N.md file is produced in the plan directory
  And it lists each task with testable acceptance criteria
  And execution does not start until the contract file exists

Scenario: Red-Green pairs distinguished in sprint contract
  Given a sprint contract for a batch containing test+impl pairs
  When the contract acceptance criteria are read
  Then test tasks show: done = "tests written and failing for the right reason (Red state)"
  And impl tasks show: done = "all tests pass (Green state, exit code 0)"

Scenario: Ambiguous criteria flagged during contract negotiation
  Given a task with BDD scenario containing vague Then-clauses
  When the evaluator reviews the acceptance criteria
  Then ambiguous criteria are flagged with suggested concrete alternatives
  And maximum 2 negotiation rounds before unresolved items escalate to user
  And unresolved items are marked as "user-accepted risk"
```
