# PRD 验证清单

生成 PRD 后，按以下步骤逐项验证文档质量。

---

## 1. 完整性验证

根据 PRD 类型检查必需章节是否齐全：

**精简版必需章节：**
- 问题陈述
- 成功指标
- 核心功能
- 时间线

**完整版必需章节（精简版 + 以下）：**
- 背景
- 目标用户
- 用户故事
- 不包含范围
- 干系人
- 风险依赖
- 待解决问题

---

## 2. SMART 目标验证

逐项检查目标是否符合 SMART 原则：

| 维度 | 检查要点 |
|------|---------|
| 具体性 | 目标是否具体明确？ |
| 可衡量 | 目标是否可量化衡量？ |
| 可实现 | 目标是否可实现？ |
| 相关性 | 目标是否与业务相关？ |
| 有时限 | 目标是否有明确时间限制？ |

不符合 SMART 原则时，提供改进建议并询问用户是否采纳。

---

## 3. 内容质量检查

| 检查项 | 标准 |
|--------|------|
| 问题陈述 | 清晰且有数据支持 |
| 成功指标 | 具体可衡量 |
| 功能描述 | 完整且可执行 |
| 时间线 | 合理可行 |
| 一致性 | 无明显遗漏或矛盾 |

发现问题时提供具体改进建议，不重新生成整个文档。

---

## 3.5 BDD 验收标准检查

检查 P0 功能的用户故事是否包含 BDD 格式的验收标准：

- 每个 P0 用户故事至少有 1 条 **假设/当/则** 验收标准
- 验收标准可测试、可验证，无模糊表述
- 边界情况和异常路径有对应的验收标准

如果缺少 BDD 验收标准，为 P0 功能自动补充建议并询问用户确认。

---

## 3.6 AI 代理可消费性检查

仅在完整版 PRD 中检查：

| 检查项 | 标准 |
|--------|------|
| 需求格式 | 每条需求为离散的、可验证的列表项 |
| 非目标表述 | 使用"禁止实现"正向约束，而非仅列表 |
| P0 阶段拆分 | 每阶段 5-15 分钟工作量，带可测试检查点 |
| 行为边界 | 包含 自主执行 / 需确认 / 硬性禁止 三层框架 |

---

## 4. 文件保存规范

**文件名格式：** `PRD-[产品名称]-[YYYYMMDD].md`

- 示例：`PRD-智能会议助手-20260128.md`

**保存路径优先级：**
1. 项目中已有的 `docs/` 或 `prd/` 目录
2. 当前工作目录

**保存后确认：**
- 报告文件保存路径
- 提供文件摘要（章节数、总字数）
