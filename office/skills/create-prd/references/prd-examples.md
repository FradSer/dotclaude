## 1. 概述

### 1.1 产品/功能简介
智能会议助手是一个面向 B2B SaaS 销售团队的 AI 驱动工具，能够自动记录会议内容、生成会议纪要、提取行动项，并与 CRM 系统集成。核心价值是将销售人员在会议后的整理时间从平均 30 分钟减少到 5 分钟以内，让销售团队更专注于客户关系建立。

### 1.2 文档目标
本 PRD 定义智能会议助手 v1.0 的产品需求，指导设计、开发和测试团队在 Q1 2026 完成 MVP 版本并在 Q2 2026 正式发布。

---

## 2. 背景与上下文

### 2.1 业务背景
公司目标在 2026 年将 SaaS 产品的客户数从 500 家增长到 1200 家，销售效率是关键制约因素。目前销售团队在会议后管理工作（整理纪要、更新 CRM、跟进行动项）上花费大量时间，导致每个销售人员每天只能安排 3-4 个客户会议，远低于行业标杆的 6-8 个。

### 2.2 市场环境
会议记录和 AI 助手市场快速增长，主要竞品包括：
- **Otter.ai**: 专注语音转文字，但缺乏 CRM 集成
- **Gong.io**: 功能强大但价格高昂（$1200/用户/年），目标客户是大企业
- **Fireflies.ai**: 功能全面，但用户界面复杂，学习成本高

**市场机会**: 针对中小 B2B SaaS 公司，提供简单易用、价格适中、与 CRM 深度集成的会议助手。

### 2.3 用户反馈/数据支持
- **用户调研（2025-Q4，N=85 销售人员）**:
  - 92% 的销售人员表示会议后整理工作"非常耗时"
  - 68% 的销售人员使用过会议记录工具，但只有 23% 持续使用（主要原因：工具不够智能、需要手动整理）
  - 79% 的销售人员表示愿意为"能直接同步到 CRM 的会议助手"付费

- **数据分析**:
  - 销售人员平均每天参加 3.2 个客户会议
  - 每次会议后平均花费 28 分钟整理纪要和更新 CRM
  - CRM 更新延迟平均 4.3 小时，导致信息不及时

## 3. 问题陈述

### 3.1 核心问题
B2B SaaS 销售团队在客户会议后需要花费大量时间（平均 28 分钟/会议）进行手动整理和 CRM 更新工作。这导致：
- **Who（谁）**: 使用 CRM 的 B2B SaaS 销售团队（目标客户群体约 500-1000 人规模的公司）
- **What（什么）**: 会议后整理纪要、提取行动项、更新 CRM 的工作耗时且重复
- **When（何时）**: 每次客户会议结束后
- **Where（何地）**: 在 Zoom/Google Meet 等线上会议工具和 Salesforce/HubSpot 等 CRM 系统之间
- **Why（为何）**: 这是销售流程的必需环节，但手动操作效率低下，容易出错
- **How（如何）**: 目前销售人员手动听录音或回忆会议内容，然后在 CRM 中逐条输入

### 3.2 问题影响
- **时间成本**: 每个销售人员每天损失约 1.5 小时（3.2 会议 × 28 分钟）在重复性工作上
- **机会成本**: 如果将这 1.5 小时用于客户会议，可以增加 50% 的会议量（从 3.2 到 4.8 个/天）
- **信息质量**: 手动整理导致约 35% 的关键信息遗漏或延迟（根据 CRM 审计数据）
- **业务影响**: 销售周期延长 15-20 天（部分原因是跟进不及时）

## 4. 目标用户与画像

### 4.1 主要目标用户

**用户画像 1: 企业销售经理**
- **基本信息**:
  - 年龄: 28-40 岁
  - 职业: Account Executive / Sales Manager
  - 行业: B2B SaaS 公司
  - 公司规模: 50-500 人
- **使用场景**:
  - 每天参加 3-5 个客户会议（线上为主）
  - 使用 Salesforce 或 HubSpot 作为 CRM
  - 需要向销售总监汇报客户进展
- **痛点**:
  - 会议后整理纪要占用大量时间
  - CRM 更新不及时导致信息滞后
  - 行动项跟踪困难，容易遗漏
- **目标**:
  - 提高会议效率，有更多时间与客户互动
  - 自动化重复性工作
  - 提升 CRM 数据质量和及时性
- **技术能力**: 中级（熟悉 SaaS 工具，能快速上手新应用）

**用户画像 2: 销售运营专员**
- **基本信息**:
  - 年龄: 25-35 岁
  - 职业: Sales Operations Specialist
  - 行业: B2B SaaS 公司
  - 公司规模: 50-500 人
- **使用场景**:
  - 负责管理销售流程和工具
  - 监控 CRM 数据质量
  - 为销售团队提供工具支持
- **痛点**:
  - 销售人员经常不及时更新 CRM
  - 难以追踪销售活动的真实情况
  - 需要手动整理销售数据做报表
- **目标**:
  - 提高 CRM 数据完整性和准确性
  - 自动化数据收集和分析
  - 减少销售人员的工具使用摩擦
- **技术能力**: 高级（熟悉各种销售工具和 API 集成）

### 4.2 次要目标用户
- **客户成功经理**: 需要记录客户沟通和续约讨论
- **产品经理**: 需要记录用户反馈会议

**目标 1**: 提升销售会议后处理效率

- **M (Measurable)**: 通过工具内置计时器和用户调研数据衡量
- **A (Achievable)**: 基于现有 AI 技术（GPT-4、Whisper）可实现 80-90% 的自动化
- **R (Relevant)**: 直接支持公司 2026 年销售效率提升 OKR
- **T (Time-bound)**: 在产品正式发布后的 3 个月内（2026 Q3）达成

**目标 2**: 提高 CRM 数据质量和及时性
- **S (Specific)**: 将 CRM 更新延迟从平均 4.3 小时降低到 1 小时以内，关键信息遗漏率从 35% 降低到 10% 以下
- **M (Measurable)**: 通过 CRM 时间戳和数据完整性审计衡量
- **A (Achievable)**: 自动同步和 AI 提取可显著减少人工干预
- **R (Relevant)**: 支持销售团队决策质量提升和销售周期缩短
- **T (Time-bound)**: 在产品正式发布后的 6 个月内（2026 Q4）达成

## 6. 用户故事与场景

### 6.1 用户故事 1: 自动会议记录
**作为** 企业销售经理
**我想要** 在会议进行时自动记录和转录内容
**以便** 我可以专注于与客户对话，而不是做笔记

**验收标准**:
- [ ] 会议开始后，工具自动加入并开始录音（需用户一次性授权）
- [ ] 实时显示语音转文字内容，准确率 ≥ 95%（中英文混合场景）
- [ ] 支持 Zoom、Google Meet、Microsoft Teams
- [ ] 区分不同发言人（至少支持 2-5 人会议）
- [ ] 会议结束后 2 分钟内生成完整转录文本

### 6.2 用户故事 2: 智能会议摘要
**作为** 企业销售经理
**我想要** 自动生成结构化的会议摘要
**以便** 我可以快速回顾会议要点，无需重听录音

**验收标准**:
- [ ] 自动识别并提取关键讨论点（3-5 条）
- [ ] 自动识别客户痛点和需求
- [ ] 自动识别客户的疑问和反对意见
- [ ] 自动提取下一步行动项（包括责任人和截止日期）
- [ ] 生成的摘要简洁明了（200-300 字）

### 6.3 用户故事 3: 自动 CRM 同步
**作为** 企业销售经理
**我想要** 会议信息自动同步到 CRM
**以便** 我不需要手动复制粘贴，节省时间并避免遗漏

**验收标准**:
- [ ] 会议结束后 5 分钟内自动创建 CRM 活动记录
- [ ] 自动关联正确的客户/联系人（基于会议参与者邮箱）
- [ ] 自动填充会议摘要、行动项等关键字段
- [ ] 支持用户在同步前预览和编辑
- [ ] 同步失败时有明确提示和重试机制

### 6.4 典型使用场景

**场景 1: 客户需求发现会议**
1. 销售经理在 Zoom 中安排会议，邀请智能会议助手
2. 会议开始，助手自动加入并开始录音转录
3. 会议进行中，销售经理专注于提问和倾听，助手实时转录
4. 会议结束后，销售经理在手机上收到通知：会议摘要已生成
5. 销售经理花 2 分钟审阅摘要和行动项，点击"同步到 CRM"
6. CRM 中自动创建活动记录，包含会议摘要、客户痛点、下一步行动
7. 销售经理立即进入下一个会议，无需花 30 分钟整理上一个会议

## 7. 功能需求

### 7.1 功能优先级说明
- **P0 (必须有)**: 没有这些功能产品无法发布
- **P1 (应该有)**: 重要功能，强烈建议包含
- **P2 (可以有)**: 次要功能，资源允许时实现
- **P3 (未来有)**: 未来版本考虑的功能

### 7.2 功能清单

#### 7.2.1 会议录制与转录

**功能 1.1: 自动会议录制**
- **优先级**: P0
- **描述**: 自动加入线上会议并录制音频
- **用户价值**: 无需手动操作，确保不会错过任何会议内容
- **功能要点**:
  - 支持 Zoom、Google Meet、Microsoft Teams
  - 会议邀请中添加助手即可自动加入
  - 录制仅限音频（视频可选，P2）
  - 支持用户设置"自动录制白名单"（特定客户或会议类型）
- **交互说明**:
  - 用户在日历中添加会议助手邮箱作为参与者
  - 会议开始时，助手显示为"会议助手（录制中）"
  - 用户可在会议中随时暂停/恢复录制
- **边界情况**:
  - 会议时长超过 3 小时：自动分段录制
  - 网络中断：自动重连，记录中断时间点
  - 会议取消/改期：自动取消录制任务
- **依赖**: Zoom/Google/Microsoft API 集成

**功能 1.2: 实时语音转文字**
- **优先级**: P0
- **描述**: 将会议语音实时转换为文字
- **用户价值**: 方便会议中快速查找信息，无需重听录音
- **功能要点**:
  - 中英文混合场景支持，准确率 ≥ 95%
  - 自动识别发言人（2-5 人会议）
  - 实时显示转录文本（延迟 < 3 秒）
  - 自动添加时间戳（每 30 秒或发言人切换时）
- **交互说明**:
  - 会议进行中，用户可在侧边栏查看实时转录
  - 点击任意文本可跳转到对应音频时间点
  - 支持关键词搜索和高亮
- **边界情况**:
  - 多人同时说话：标记为"[重叠发言]"
  - 背景噪音大：自动降噪，标记低置信度内容
  - 非中英文语言：显示提示"不支持的语言"
- **依赖**: Whisper API 或自建语音识别服务

**功能 1.3: 发言人识别**
- **优先级**: P1
- **描述**: 区分会议中的不同发言人
- **用户价值**: 清晰了解谁说了什么，方便后续跟进
- **功能要点**:
  - 自动识别 2-5 个发言人
  - 根据会议参与者列表自动匹配姓名
  - 支持用户手动修正发言人标签
- **交互说明**:
  - 转录文本中以"张三: [内容]"格式显示
  - 用户可点击发言人名称查看其所有发言
- **边界情况**:
  - 发言人超过 5 人：仅标记"发言人 A/B/C..."
  - 无法匹配参与者：显示"未知发言人"，支持手动标记
- **依赖**: 声纹识别技术

#### 7.2.2 智能内容提取

**功能 2.1: 会议摘要生成**
- **优先级**: P0
- **描述**: AI 自动生成结构化的会议摘要
- **用户价值**: 快速了解会议核心内容，无需阅读完整转录
- **功能要点**:
  - 摘要包含：会议主题、关键讨论点（3-5 条）、重要决策
  - 长度控制在 200-300 字
  - 使用要点列表格式，易于扫读
  - 生成时间 < 2 分钟（会议结束后）
- **交互说明**:
  - 会议结束后自动生成，通过通知提醒用户
  - 用户可点击"重新生成"调整摘要风格（简洁/详细）
  - 支持手动编辑摘要内容
- **边界情况**:
  - 会议时长 < 5 分钟：不生成摘要，直接显示转录
  - 会议内容无关（如闲聊）：显示"本次会议未检测到业务讨论内容"
- **依赖**: GPT-4 API

**功能 2.2: 行动项提取**
- **优先级**: P0
- **描述**: 自动识别并提取会议中的行动项（Action Items）
- **用户价值**: 确保后续跟进，避免遗漏关键任务
- **功能要点**:
  - 自动识别行动项关键词（如"需要"、"跟进"、"准备"、"发送"）
  - 提取责任人（如有明确提及）
  - 提取截止日期（如有明确提及）
  - 每个行动项包含上下文引用（原始对话片段）
- **交互说明**:
  - 显示为清单格式，每项可勾选完成状态
  - 点击行动项可查看原始对话上下文
  - 支持手动添加/编辑/删除行动项
- **边界情况**:
  - 未明确责任人：标记为"待分配"
  - 未明确截止日期：标记为"待确认"
  - 模糊的行动项（如"再看看"）：标记"需要明确"
- **依赖**: GPT-4 API

**功能 2.3: 客户洞察提取**
- **优先级**: P1
- **描述**: 自动识别客户的痛点、需求、疑问和反对意见
- **用户价值**: 帮助销售人员更好理解客户需求，制定针对性策略
- **功能要点**:
  - 识别客户明确表达的痛点
  - 识别客户的核心需求和期望
  - 识别客户的疑问和关注点
  - 识别客户的反对意见或顾虑
- **交互说明**:
  - 分类显示（痛点/需求/疑问/反对意见）
  - 每条洞察关联原始对话片段
  - 支持添加标签和备注
- **边界情况**:
  - 客户未明确表达：标记"需要进一步确认"
- **依赖**: GPT-4 API

#### 7.2.3 CRM 集成

**功能 3.1: 自动 CRM 同步**
- **优先级**: P0
- **描述**: 将会议信息自动同步到 CRM 系统
- **用户价值**: 消除手动数据输入，提高 CRM 数据及时性和准确性
- **功能要点**:
  - 支持 Salesforce、HubSpot
  - 自动创建"会议"活动记录
  - 自动关联客户/联系人（基于会议参与者邮箱）
  - 自动填充字段：会议日期、时长、摘要、行动项
  - 支持自定义字段映射
- **交互说明**:
  - 会议结束后显示"同步到 CRM"按钮
  - 同步前显示预览页面，用户可编辑内容
  - 同步后显示成功提示，包含 CRM 记录链接
- **边界情况**:
  - CRM 中找不到匹配客户：提示用户手动选择或创建新客户
  - CRM API 调用失败：显示错误信息，支持重试
  - 重复同步：检测已存在记录，询问是否覆盖或追加
- **依赖**: Salesforce/HubSpot API 集成

**功能 3.2: 行动项转任务**
- **优先级**: P1
- **描述**: 将提取的行动项转换为 CRM 任务
- **用户价值**: 确保行动项进入正式跟进流程，避免遗忘
- **功能要点**:
  - 自动将行动项创建为 CRM 任务
  - 自动设置责任人、截止日期（如有）
  - 任务描述包含原始会议上下文
  - 任务关联对应的客户/联系人
- **交互说明**:
  - 在同步 CRM 时，勾选"同时创建任务"
  - 可批量选择需要创建任务的行动项
  - 支持设置任务提醒
- **边界情况**:
  - 行动项无责任人：任务分配给会议组织者
  - 行动项无截止日期：设置默认截止日期（如会议后 7 天）
- **依赖**: Salesforce/HubSpot Task API

#### 7.2.4 用户管理

**功能 4.1: 用户认证与授权**
- **优先级**: P0
- **描述**: 用户登录和权限管理
- **用户价值**: 保障账户安全和数据隐私
- **功能要点**:
